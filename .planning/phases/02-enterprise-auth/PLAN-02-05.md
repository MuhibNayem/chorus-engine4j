# Plan 02-05: Security Integration & Tests

## Goal
Write integration tests for all Phase 2 auth subsystems: OAuth2 JIT provisioning, SAML assertion replay prevention, SCIM token authentication, and SCIM user CRUD operations. All tests use hand-written in-memory fakes (no Mockito, no @SpringBootTest) following the Phase 1 test pattern.

## must_haves
- All new repositories have in-memory fake implementations for testing.
- `JitProvisioningService` tests cover: new user JIT, existing user link, role preservation, default role assignment.
- `ScimTokenAuthFilter` tests cover: valid token, expired token, revoked token, missing header, wrong path bypass.
- `ScimUserController` tests (via MockHttpServletRequest/Response) cover: POST create, GET by id, GET list with filter, PUT update, DELETE soft-deactivate, duplicate email 409.
- `AssertionIdCache` tests cover: first assertion accepted, duplicate within 2 min rejected, duplicate after 2 min accepted (requires time manipulation or short TTL for test).
- `./gradlew :chorus-observe-server:test` passes with all new tests green.

<action>
Create three in-memory fake repository files:

**InMemoryScimTokenRepository.java:**
- Extends nothing, implements the same public methods as `ScimTokenRepository`.
- Uses a `ConcurrentHashMap<String, ScimToken>` keyed by `tokenHash`.
- Methods: `save`, `findById`, `findByTokenHash`, `findByTenantId`, `revokeById`, `deleteById`.
- `findByTenantId` filters by tenantId.
- `revokeById` sets `revokedAt = Instant.now()`.

**InMemoryTenantOauthConfigRepository.java:**
- Uses `ConcurrentHashMap<UUID, TenantOauthConfig>` keyed by id.
- Also maintains a `Map<String, TenantOauthConfig>` for `(tenantId + "::" + providerName)` lookup.
- Methods: `save`, `findById`, `findByTenantId`, `findByTenantIdAndProviderName`, `findEnabledByTenantId`, `deleteById`.

**InMemoryTenantSamlConfigRepository.java:**
- Same pattern as InMemoryTenantOauthConfigRepository.
- Methods: `save`, `findById`, `findByTenantId`, `findByTenantIdAndProviderName`, `findEnabledByTenantId`, `deleteById`.

**Constructor pattern:** Follow `InMemoryApiKeyRepository` — accept `DataSource` parameter but ignore it (store in memory).

**JSONB handling:** For `scopes` (List<String>), store directly as `List<String>` in memory; no JSON serialization needed.
</action>

## Task 4: Write SCIM user controller tests

<read_first>
- chorus-observe-server/src/main/java/com/chorus/observe/api/scim/ScimUserController.java
- chorus-observe-server/src/main/java/com/chorus/observe/service/ScimUserService.java
</read_first>

<action>
Create `chorus-observe-server/src/test/java/com/chorus/observe/api/scim/ScimUserControllerTest.java`:

Write 6 tests. Since we can't use `@SpringBootTest` or `MockMvc`, test the controller and service layer directly:

**Approach:** Instantiate `ScimUserController` with a `ScimUserService` backed by `InMemoryUserRepository`, `InMemoryRoleRepository`, `InMemoryUserRoleRepository`. Set `TenantContext` before each test and clear after.

1. `shouldCreateUser()` — POST equivalent: call `scimUserController.create(dto)`. Assert 201 status, returned DTO has generated id, `userName` matches.

2. `shouldReturn409OnDuplicateEmail()` — Create a user first, then call create with same email. Assert response is 409 Conflict with `ScimErrorResponse`.

3. `shouldGetUserById()` — Create a user, then call `get(id)`. Assert 200, email matches.

4. `shouldReturn404ForMissingUser()` — Call `get("nonexistent")`. Assert 404.

5. `shouldListUsersWithFilter()` — Create 2 users. Call `list` with `filter="userName eq \"alice@corp.com\""`. Assert only alice is returned, `totalResults=1`.

6. `shouldSoftDeleteUser()` — Create a user, call `delete(id)`. Assert 204. Then verify via `InMemoryUserRepository` that the user's status is `INACTIVE`.

**Note:** Since `ScimUserController` returns `ResponseEntity`, we can directly call the methods and assert on the returned entities. No need for MockMvc.
</action>

- Tests use `InMemoryUserRepository`, `InMemoryRoleRepository`, `InMemoryUserRoleRepository`.
- Tests set and clear `TenantContext` (use `TenantContext.set(tenantId, null, null)` / `TenantContext.clear()`).
